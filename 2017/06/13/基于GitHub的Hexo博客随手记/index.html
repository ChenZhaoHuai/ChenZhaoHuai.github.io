基于GitHub的Hexo博客随手记

发表于 2017-06-13 | 阅读次数

第一次搭建博客，在用Mac搭建Hexo博客于Github上的完整历程，写一些东西记录下。

安装前提

安装Hexo前，必须保证电脑中是否已安装以下前提：

  • Node.js
  • Git

安装步骤就不再此详细说明，安装官网的步骤操作即可。

安装Hexo

使用命令行安装Hexo

$ npm install hexo-cli -g

创建一个文件夹初始化blog。

Next主题安装

Next主题安装配置，根据Next文档中介绍安装配置Next主题。

Hexo简单使用命令

这里介绍几个Hexo常用的命令:

  1. hexo generate (hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹
  2. hexo server (hexo s) 启动本地web服务，用于博客的预览
  3. hexo deploy (hexo d) 部署播客到远端（比如github, heroku等平台）

新建一篇文章：

$ hexo new 基于GitHub的Hexo博客随手记

在blog/source/_posts/目录下修改基于GitHub的Hexo博客随手记.md文件，然后执行以下命令生成html：

hexo generate

绑定域名

比较了阿里的万网和腾讯的腾讯云发现我的域名==chenzhaohuai.com==在腾讯的腾讯云比较便宜，所以在腾讯云购买域名==chenzhaohuai.com==，以==blog.chenzhaohuai.com==作为博客的域名。
按以下步骤进行配置域名解析。

  1. 进入腾讯云的余名管理后台
  2. 添加CNAME记录类型记录值对应你gitHub的博客域名即==chenzhaohuai.github.io.== 主机记录默认@
  3. 添加记录类型A主机记录为blog，对应的记录值是你gitHub博客的ip地址。

如以下图所示

在gitHub你的博客路径下创建一个CNAME文件，里面填写你的域名==blog.chenzhaohuai.com==

遇到的坑

  1. 使用搭建Hexo命令时，出现==Permission denied==问题，猜测可能是权限问题，所以执行==sudo -s==命令获取权限。
  2. CNAME文件创建问题，CNAME一定要在gitHub创建，然后下载下来，然后在复制道==blog/public/==路径下，不然每次同步道gitHub会被覆盖掉文件。
